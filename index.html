CLAUDE.md とは

プロジェクトのルール・規約をまとめた取扱説明書。Claude Codeが作業のたびに必ず読み込む

🚨

なぜ必要?

Slackくらいの規模になると、毎回「Tailwind使って」「日本語で」と指示するのは限界。AIは前の会話を忘れる

✅

効果

デザインやコード規約がブレなくなる。新しいページを作っても統一感が保たれる

CLAUDE.md

# このプロジェクトについて

Slack風のチャットアプリを作っています。Next.js + Tailwind CSS + Supabaseで構築します。

# コーディング規約

- TypeScript を使う - コンポーネントは src/components/ に配置 - ページは src/app/ 配下（App Router） - スタイリングは Tailwind CSS のみ。CSSファイルは作らない - コメントは日本語で書く

# デザインルール

- ベースカラーは白、アクセントは Slack 風の紫 (#4A154B) - フォントは system-ui を使う - 角丸は rounded-lg で統一 - ボタンは必ずホバー時に色が変わるようにする

# やってほしくないこと

- 勝手にライブラリを追加しない（必要なら先に確認する） - インラインstyleを使わない - any 型を使わない
SESSION 2

チャットページを作るプロンプト

PROMPT

# 目的

Slackのチャットページを作ってください。既に作った全体レイアウトの中の、真ん中のチャットエリアを作り込みます。

# レイアウト

- 上部: チャンネル名（例 #general）、メンバー数、検索アイコン - 中央: メッセージ一覧（上から下に時系列で表示、下が最新） - 下部: メッセージ入力フォーム（固定）

# メッセージの表示

- 左にユーザーアバター（丸、40px） - 右にユーザー名（太字）+ タイムスタンプ（小さく薄く） - その下にメッセージ本文 - 同じユーザーが連投した場合はアバター・名前を省略してまとめる - ホバーでリアクション追加・返信ボタンが表示される

# 入力フォーム

- 複数行入力できるテキストエリア - 左に「＋」ファイル添付ボタン、右に「送信」ボタン - Enterで送信、Shift+Enterで改行 - 下に「@メンション」「絵文字」のUIを置く

# ダミーデータ

- 10件くらいのメッセージをハードコードで入れておく - ユーザーは3人分用意する - タイムスタンプは「14:32」の形式

# デザイン

- Slack本家の雰囲気（白背景、グレーのボーダー、紫のアクセント） - Tailwind CSS を使う - レスポンシブ対応（モバイルではサイドバーを畳む）

SESSION 4

TOKEN・Skills・グローバル設定

TOKEN

トークンの概念

AIが処理するテキストの単位。入力+出力トークンで課金。長い会話ほどトークン消費が多い。コンテキストウィンドウの制限を意識しよう

SKILLS

スキル機能

Claude Codeの拡張機能。/commit、/review-pr などの組み込みスキルや、カスタムスキルの作成方法

CONFIG

グローバル設定

~/.claude/settings.json でClaude Code全体の動作をカスタマイズ。フック、MCP、権限管理

MOBILE

スマホでコーディング

Claude Code Webアプリ (claude.ai/code) を使えばスマホからもコーディング可能。外出先での簡単な修正に便利

Claude Code vs Codex

⚡

Claude Code

• ターミナル/IDE/Webで動作
• ファイル編集・コマンド実行が可能
• MCP連携で外部ツール接続
• CLAUDE.mdでプロジェクト設定
• 対話的に段階的に構築

💻

OpenAI Codex

• サンドボックス環境で実行
• GitHub PR自動作成
• 非同期タスク処理
• 複数タスク並列実行
• バッチ処理向き

使い分けのポイント

Claude Code = 対話しながらリアルタイムに開発
Codex = まとめて自動処理・バッチタスク
