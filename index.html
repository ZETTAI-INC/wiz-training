# 使う技術

- データベース・認証・リアルタイム: Supabase - フロント: 既存の index.html（バニラJS）に @supabase/supabase-js を CDN で読み込む - 追加のフレームワークは入れない（前のステップの構成を壊さない）

# Supabase 側のセットアップ手順

1. Supabase ダッシュボードで新規プロジェクトを作成する手順を、最初に箇条書きで案内する 2. 取得すべき値を明示: SUPABASE_URL と SUPABASE_ANON_KEY 3. 下記の SQL を SQL Editor に貼って実行するよう指示する（テーブル作成 + RLS + Realtime 有効化までワンセット）

# SQL（このまま貼れば動く形で出力する）

- profiles テーブル: id (uuid, auth.users と紐付け), display_name (text), avatar_color (text), created_at - channels テーブル: id (uuid pk), name (text unique), description (text), created_at - messages テーブル: id (uuid pk), channel_id (uuid fk → channels), user_id (uuid fk → profiles), body (text), created_at - それぞれに enable row level security - ポリシー: ・誰でも select 可（読み取りは全公開） ・insert は auth.uid() = user_id のときだけ許可 ・update / delete も自分の行だけ - 初期データ: # general / # random / # dev / # design を insert - 最後に alter publication supabase_realtime add table messages; で Realtime を有効化

# フロント側の改修

1. <head> に Supabase JS の CDN を追加 <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script> 2. <script> 冒頭で const supabase = window.supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY) を初期化 3. キー2つはファイル先頭の // === ここに Supabase の値を入れる === ブロックに分かりやすく置く 4. ハードコードしていたメッセージ配列を削除し、起動時に messages を created_at 昇順で select して描画する関数 loadMessages() を作る 5. メッセージ送信: 入力欄で Enter を押したら supabase.from('messages').insert({...}) を呼ぶ 6. リアルタイム購読: supabase.channel('messages').on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'messages' }, payload => 描画追加).subscribe() 7. 画面下にスクロールを自動で追従させる

# 認証（最小構成でOK）

- 起動時に supabase.auth.getSession() を呼び、未ログインなら匿名でサインイン（signInAnonymously） - サインインしたら profiles に自分の行が無ければ作る（display_name はランダムな日本人名、avatar_color はランダム） - user_id は常に auth.uid() を使う

# エラーハンドリング

- Supabase の応答が error を含む場合は console.error + 画面上部に赤いトーストを2秒表示 - ネットワーク断のときは入力欄を一時的にグレーアウト

# 必須条件（これを全部満たせば「完成」）

1. 別タブで2枚開いて、片方で送信したらもう片方に自動で追加される 2. リロードしても過去メッセージが残っている 3. ダミーデータを完全に撤去している（DBの中身だけが表示される） 4. SUPABASE_URL / SUPABASE_ANON_KEY の差し替え場所が1箇所にまとまっている 5. SQLをそのまま貼れば、テーブル・RLS・Realtime まで一気に整う

# 仕上げ

- 完成したら最後に「① Supabase ダッシュボードでプロジェクト作成 → ② SQL を貼って実行 → ③ URL/KEY を index.html に貼る → ④ ブラウザで開く」の4ステップを箇条書きで添える - 途中で止まらず、最も自然な解釈で実装を進めること - 「できません」「環境がないので不明です」と言わず、必ず最後まで作り切ること

TOKEN

トークンの概念

AIが処理するテキストの単位。入力+出力トークンで課金。長い会話ほどトークン消費が多い。コンテキストウィンドウの制限を意識しよう

SKILLS

CONFIG

グローバル設定

~/.claude/settings.json でClaude Code全体の動作をカスタマイズ。フック、MCP、権限管理

MOBILE

スマホでコーディング

Claude Code Webアプリ (claude.ai/code) を使えばスマホからもコーディング可能。外出先での簡単な修正に便利
