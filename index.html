Claude Code Tips

より効果的にClaude Codeを使いこなすためのテクニック集

SESSION 4

CLAUDE.md ファイル

📜

CLAUDE.md とは

プロジェクトのルールや指示を記述するファイル。Claude Codeが毎回読み込み、プロジェクト固有の振る舞いを定義できる

🛠️

使い方

プロジェクトルートにCLAUDE.mdを配置。コーディング規約、使用ライブラリ、注意事項を記述

# CLAUDE.md の例

## プロジェクトルール
- TypeScriptを使用すること
- Tailwind CSS でスタイリング
- コンポーネントは src/components/ に配置
- 日本語でコメントを書くこと
SESSION 4

TOKEN・Skills・グローバル設定

TOKEN

トークンの概念

AIが処理するテキストの単位。入力+出力トークンで課金。長い会話ほどトークン消費が多い。コンテキストウィンドウの制限を意識しよう

SKILLS

スキル機能

Claude Codeの拡張機能。/commit、/review-pr などの組み込みスキルや、カスタムスキルの作成方法

CONFIG

グローバル設定

~/.claude/settings.json でClaude Code全体の動作をカスタマイズ。フック、MCP、権限管理

MOBILE

スマホでコーディング

Claude Code Webアプリ (claude.ai/code) を使えばスマホからもコーディング可能。外出先での簡単な修正に便利

05

Codex

OpenAI Codexとの比較から、それぞれの強みを理解する

SESSION 5

Claude Code vs Codex

⚡

Claude Code

• ターミナル/IDE/Webで動作
• ファイル編集・コマンド実行が可能
• MCP連携で外部ツール接続
• CLAUDE.mdでプロジェクト設定
• 対話的に段階的に構築

💻

OpenAI Codex

• サンドボックス環境で実行
• GitHub PR自動作成
• 非同期タスク処理
• 複数タスク並列実行
• バッチ処理向き

使い分けのポイント

Claude Code = 対話しながらリアルタイムに開発
Codex = まとめて自動処理・バッチタスク
