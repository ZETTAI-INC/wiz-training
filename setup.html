Claude Code研修
事前準備手順書

やることは全部で6つです。研修開始前に終わらせてください。

先に準備するもの

  • パソコン
  • 使えるメールアドレス 1つ
  • Chrome
  • インターネット
  • メモできるもの
GitHubもSupabaseも同じメールアドレスで登録してください。
バラバラだと、組織招待のときにかなり詰まりやすいです。
GitHubは認証済みメールが必要で、Supabaseの組織招待もメールで届きます。

1. Antigravity を入れる

開く場所
https://antigravity.google/download

公式の入手先です。macOS / Windows / Linux 向けに配布されています。

やること

  1. 上のURLを開く
  2. 自分のパソコンに合うものを選ぶ
  3. ダウンロードする
  4. インストールする
  5. 起動する

ここで確認すること

  • アプリが開く
  • ログイン画面まで行ける

つまずきやすい点

  • 学校や会社のパソコンでインストール制限がある
  • Chromeが入っていない
  • ログイン用アカウントを間違える
公式Codelabでもインストールから使い始める流れが案内されています。
公式 Codelab

2. GitHub に登録する

開く場所
https://github.com/

github.com から Sign up を押して登録します。
登録後はメール認証が必要です。認証しないと基本的な作業ができません。

やること

  1. GitHubを開く
  2. Sign up を押す
  3. メールアドレスを入れる
  4. パスワードを決める
  5. ユーザー名を決める
  6. 登録を完了する
  7. 届いたメールを開いて認証する

ユーザー名の決め方

変な名前にしないで、研修で見ても誰かわかる名前にしてください。

zettai-tanaka yamada-dev suzuki-study

ここで確認すること

  • GitHubにログインできる
  • メール認証が終わっている

つまずきやすい点

  • メール認証を忘れる
  • ユーザー名を適当に決めて後で誰かわからなくなる
  • パスワードをメモしていない

3. GitHub の組織に入る

GitHubには個人アカウントと組織アカウントがあります。
研修では、運営側の組織に参加してコードを見る流れになります。

招待が来たらやること

  1. GitHubに登録したメールアドレスを開く
  2. 招待メールを探す
  3. メールの中の参加リンクを押す
  4. GitHubで承認する

ここで確認すること

  • 組織ページが見える
  • 指定されたリポジトリが見える

メールが来ないとき

  • 迷惑メールを見る
  • GitHub登録メールが合っているか確認する
  • 運営に「GitHubユーザー名」を伝える

4. Supabase に登録する

ダッシュボード
https://supabase.com/dashboard/sign-in
公式サイト
https://supabase.com/

Supabaseは認証やデータベースを使うためのサービスです。
公式サイトからダッシュボードに入り、アカウントを作成します。

やること

  1. Supabaseを開く
  2. サインアップする
  3. メールアドレスを入れる
  4. パスワードを決める
  5. 必要ならメール認証をする
  6. ダッシュボードに入る

ここで確認すること

  • Supabaseにログインできる
  • ダッシュボード画面が見える

つまずきやすい点

  • GitHubと別のメールで登録してしまう
  • ログインしたつもりで、実は別アカウントに入っている

5. Supabase の組織に入る

組織のTeam設定から招待が届きます。
招待リンクは24時間有効です。届いたら早めに対応してください。

招待が来たらやること

  1. Supabaseに登録したメールアドレスを開く
  2. 招待メールを探す
  3. メールの中のリンクを押す
  4. 参加する
  5. 指定された組織やプロジェクトが見えるか確認する

ここで確認すること

  • 組織が見える
  • 指定されたプロジェクトが見える

メールが来ないとき

  • 迷惑メールを見る
  • 登録メールが合っているか確認する
  • 24時間を過ぎたら運営に再送してもらう

6. claude-yolo を入れる

リポジトリ
https://github.com/eastlondoner/claude-yolo

Claude CLIをラップするツールです。
YOLOモードは安全チェックを回避するため、データ削除や情報漏えいの危険があります。

最初はSAFEモードだけで使ってください。
初心者に「とりあえず全部飛ばしていい」と教えると、何が危ないのか理解しないまま進みます。
研修では SAFE 固定で十分です。

入れ方

ターミナルを開いて、以下を実行します。

npm install -g github:maxprez/claude-yolo

起動確認

# SAFEモードで起動
claude-yolo --safe

# または
claude-yolo --no-yolo

モードの確認と切り替え

# 現在のモードを確認
claude-yolo mode

# SAFEモードに切り替え
claude-yolo mode safe

ここで確認すること

  • インストールが完了した
  • SAFEモードで起動できた
