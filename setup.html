STEP 2

GitHub に登録する

開く場所
https://github.com/
→

Sign up を押して登録します。登録後はメール認証が必要です。

やること
  1. GitHubを開く
  2. Sign up を押す
  3. メールアドレスを入れる
  4. パスワードを決める
  5. ユーザー名を決める
  6. 登録を完了する
  7. 届いたメールを開いて認証する
ユーザー名の決め方

研修で見ても誰かわかる名前にしてください。

zettai-tanaka yamada-dev suzuki-study
ここで確認すること
  • GitHubにログインできる
  • メール認証が終わっている
つまずきやすい点
  • メール認証を忘れる
  • ユーザー名を適当に決めて後で誰かわからなくなる
  • パスワードをメモしていない
STEP 3

GitHub の組織に入る

GitHubには個人アカウントと組織アカウントがあります。研修では運営側の組織に参加してコードを見る流れです。

招待が来たらやること
  1. GitHubに登録したメールアドレスを開く
  2. 招待メールを探す
  3. メールの中の参加リンクを押す
  4. GitHubで承認する
ここで確認すること
  • 組織ページが見える
  • 指定されたリポジトリが見える
メールが来ないとき
  • 迷惑メールを見る
  • GitHub登録メールが合っているか確認する
  • 運営に「GitHubユーザー名」を伝える
STEP 4

Supabase に登録する

ダッシュボード
https://supabase.com/dashboard/sign-in
→
公式サイト
https://supabase.com/
→

Supabaseは認証やデータベースを使うためのサービスです。

やること
  1. Supabaseを開く
  2. サインアップする
  3. メールアドレスを入れる
  4. パスワードを決める
  5. 必要ならメール認証をする
  6. ダッシュボードに入る
ここで確認すること
  • Supabaseにログインできる
  • ダッシュボード画面が見える
つまずきやすい点
  • GitHubと別のメールで登録してしまう
  • ログインしたつもりで、実は別アカウントに入っている
STEP 5

Supabase の組織に入る

組織のTeam設定から招待が届きます。招待リンクは24時間有効です。

招待が来たらやること
  1. Supabaseに登録したメールアドレスを開く
  2. 招待メールを探す
  3. メールの中のリンクを押す
  4. 参加する
  5. 指定された組織やプロジェクトが見えるか確認する
ここで確認すること
  • 組織が見える
  • 指定されたプロジェクトが見える
メールが来ないとき
  • 迷惑メールを見る
  • 登録メールが合っているか確認する
  • 24時間を過ぎたら運営に再送してもらう
STEP 6

claude-yolo を入れる

リポジトリ
https://github.com/eastlondoner/claude-yolo
→

Claude CLIをラップするツールです。YOLOモードは安全チェックを回避するため、データ削除や情報漏えいの危険があります。

最初はSAFEモードだけで使ってください。
初心者に「とりあえず全部飛ばしていい」と教えると、何が危ないのか理解しないまま進みます。研修ではSAFE固定で十分です。
入れ方
npm install -g github:maxprez/claude-yolo
起動確認
# SAFEモードで起動
claude-yolo --safe

# または
claude-yolo --no-yolo
モードの確認と切り替え
# 現在のモードを確認
claude-yolo mode

# SAFEモードに切り替え
claude-yolo mode safe
ここで確認すること
  • インストールが完了した
  • SAFEモードで起動できた
